## 经验教训

### 已验证的决策
1. **Flask + 原生前端**：适合初学者，哲哥能理解和修改
2. **内存数据模型**：启动快，无数据库配置，适合演示
3. **单文件 app.py**：初期方便，后期维护困难（1.2万行）

### 需要改进的决策
1. **单文件膨胀**：app.py 1.2万行，应尽早拆分 routes/
2. **缺乏自动化测试**：无 pytest，迭代风险累积
3. **缺乏 CI/CD**：无 GitHub Actions，部署手动化
4. **数据持久化**：内存数据重启丢失，需 SQLite 持久化或导出机制
