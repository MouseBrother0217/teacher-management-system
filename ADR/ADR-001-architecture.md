# ADR-001: 师资管理系统架构决策

## 状态
- 状态: 提案中
- 日期: 2026-04-16
- 决策者: 哲哥（需求方）+ 鼠小弟（技术执行）

---

## 背景

哲哥（软件开发小白）希望开发一款Web版师资管理系统，用于管理教师信息、课程安排、教学评估等。

这是第一个实战项目，需要在实践中学习软件开发全流程。

---

## 决策

### 1. 技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | HTML + CSS + JavaScript (原生) | 小白友好，无需复杂框架，先掌握基础 |
| 后端 | Python + Flask | 轻量级，文档丰富，适合初学者 |
| 数据库 | SQLite | 零配置，单文件，快速上手 |
| 部署 | 本地运行 → 后续Docker化 | 先跑通，再优化 |

### 2. 功能范围（MVP）

**第一阶段（核心功能）:**
- [ ] 教师信息管理（增删改查）
- [ ] 课程管理（增删改查）
- [ ] 教师-课程关联
- [ ] 简单的Web界面

**第二阶段（扩展功能）:**
- [ ] 教学评估系统
- [ ] 数据统计报表
- [ ] 用户权限管理

### 3. 架构模式

采用 **MVC模式**:
- **Model**: 数据库模型（SQLAlchemy）
- **View**: HTML模板（Jinja2）
- **Controller**: Flask路由处理

### 4. 项目结构

```
teacher-management-system/
├── app.py              # 应用入口
├── config.py           # 配置文件
├── models/             # 数据模型
│   ├── __init__.py
│   ├── teacher.py
│   └── course.py
├── routes/             # 路由控制器
│   ├── __init__.py
│   ├── teacher_routes.py
│   └── course_routes.py
├── templates/          # HTML模板
│   ├── base.html
│   ├── teachers/
│   └── courses/
├── static/             # CSS/JS/图片
│   ├── css/
│   └── js/
├── tests/              # 测试文件
│   └── test_basic.py
├── requirements.txt    # 依赖列表
└── README.md           # 项目说明
```

## 替代方案考虑

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| Django | 功能完整，内置admin | 学习曲线陡峭，概念多 | ❌ 放弃，对小白太复杂 |
| FastAPI | 现代，自动文档 | 需要理解async概念 | ❌ 放弃，先掌握基础 |
| **Flask** | 简单，灵活，文档友好 | 需要手动配置较多 | ✅ 选择，适合学习 |
